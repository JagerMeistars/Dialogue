# dcs Dialog System

Диалоговая система для Minecraft-карт (проверено на 26.3-snapshot-4,
datapack pack_format 111): панельные окна
реплик NPC поверх actionbar, иконки персонажей, цвета имён, плавные
фейды, двухстрочные окна, пословные текст-эффекты (дрожь/глитч/пульс/прыжки),
случайные наборы реплик и полная локализация (ru/en). Ванильная: только
ресурспак + датапак, без модов.

Сделано студией DreamCloudsStudio (DCS) для карты «Обычный случай UPDATE»
(коллаборация с qwer team).

## Как это устроено (в двух словах)

- Окно диалога — это обычный **actionbar** (`title @s actionbar [...]`), но
  собственный **core-шейдер** `text.vsh/fsh` распознаёт «магические»
  цвета текста и превращает их в графику:
  - компонент `{"text":".","color":"#DCGGBB"}` — маркер: точка растягивается
    шейдером в панель-подложку под всей строкой (ширина берётся из симметрии
    actionbar относительно центра экрана, NDC конца строки = −NDC начала);
  - `#FBGGBB` — имя/первый ряд: палитра NPC, волна, ступень фейда;
  - `#F9GGBB` — второй ряд: рисуется на 12px ниже (перекрашивается в белый);
  - `#FAGGBB` — эффект-текст: дрожь, глитч, пульс, прыжки (работает и в GUI, и
    на мировом `text_display`).
  Полная битовая раскладка задокументирована в шапке `text.vsh`.
  Весь остальной текст игры рендерится ванильно.
- **Плавность**: фейд = 8 ступеней яркости/альфы, обычные тики actionbar
  просто перерисовывают окно нужной ступенью (см. генерируемые `*_lines`).
- **Двухстрочность + локализация**: горизонтальную вёрстку второй строки шейдер
  сделать не может (не знает ширин текста), поэтому сдвиги зашиваются прямо в
  **lang-значение** невидимыми глифами-спейсерами (шрифт `dcs:offsets`:
  E100..E108 = −1..−256px, E110..E118 = +1..+256px). Значение второй строки =
  `[спейсер −ширина ряда 1][текст][добивка до ширины объединения]`.
  Ширины генератор меряет по PNG шрифтов **на каждый язык отдельно**, поэтому
  вёрстка верна и в ru, и в en.
- **Иконки NPC** — шрифт `dcs:icons` (E001.. реплики; height 16,
  ascent 11, advance 17px — ширина учитывается генератором).
- Все тексты — `translate`-компоненты, ключи и lang-файлы генерирует Python.

## Состав пакета

```
gen_dialogs.py      генератор: КОНТЕНТ РЕПЛИК + вся сборка (окна, lang, роутинг)
mcfont/             ванильные шрифты из client.jar (для замера ширин глифов)
resourcepack/       шейдер, шрифты (offsets/icons), оверрайд default.json, lang
datapack/           dialog/begin|tick|end (генерируются), npc/dialog/*,
                    минимальные load/tick
```

Текущий контент (5 NPC карты «Обычный случай UPDATE») оставлен как рабочий
пример — заменяется на своих персонажей.

## Установка на новую карту

1. Содержимое `resourcepack/assets` влить в ресурспак карты, `datapack/data` —
   в датапак (или бросить оба пакета как есть: mcmeta уже лежат).
   Если в датапаке уже есть свои `load`/`tick` — перенести строки из наших
   (нужны только objectives `dcs.dlg`, `dcs.data` и один тиковый хук).
2. В `gen_dialogs.py` сверху — блок путей: по умолчанию он пишет в
   `resourcepack/` и `datapack/` внутри пакета; можно указать абсолютные пути
   прямо в карту.
3. Отредактировать контент (см. ниже), запустить `python gen_dialogs.py`
   (нужен Pillow: `pip install pillow`).
4. В игре: `F3+T` (ресурспак) + `/reload` (датапак).
5. Запуск реплики: `/function dcs:npc/dialog/<npc>` (случайный набор) или
   `/function dcs:npc/dialog/<npc>_<набор>` (конкретный). Вызывается как игрок
   (обычно из клика по interaction-хитбоксу:
   `execute as @e[...] on target run function ...`).

## Как писать контент

Всё — в `gen_dialogs.py`, словарь `npcs`.

```python
npcs = {
    "имя_ключ": (ид_палитры, "Отображаемое имя", "звук.голоса", питч, наборы),
}
```

- `ид_палитры` 1..5 — цвет имени и иконка (E00N). Цвета палитры — в
  `dcsPalette()` в `text.vsh`, там же их менять.
- `наборы` — либо список окон (одна последовательность), либо dict
  `{"вариант": [окна...]}` — тогда генератор делает `/random`-диспетчер и клик
  выдаёт случайный набор.
- Окно = `(ru, en)`. Строка — одна (`"текст"`) или две (`("ряд 1", "ряд 2")`).
- Эффект на часть текста — список сегментов:
  `["обычный ", ("трясущийся", "shake")]`. Эффекты: `shake`, `glitch`,
  `pulse`, `bouncy`. Эффект на всё окно — третьим элементом кортежа.
- Длительность окна считается сама: `25 + 1.3×длина`, зажата в 45..140 тиков.
- Реакции `hits` (реплика при попадании предметом и т.п.) — отдельный блок,
  в случайные диспетчеры не входят, дёргаются напрямую:
  `function dcs:npc/dialog/<npc>_hit`.
- Прочие ключи локализации карты (названия предметов и т.п.) можно класть в
  блок «прочие ключи» в конце — тогда они переживут регенерацию.

**ВАЖНО: lang-файлы пишутся генератором С НУЛЯ.** Ручные правки
`ru_ru.json`/`en_us.json` сотрутся — весь текст живёт только в генераторе.
Флаг `EN_MIRROR_RU = True` делает en_us копией ru_ru (для русских игроков на
английской локали); английские тексты при этом сохраняются в контенте на будущее.

## Добавление нового NPC (чек-лист)

1. Нарисовать иконку 16×16 → `resourcepack/assets/dcs/textures/font/`.
2. Прописать её в `resourcepack/assets/dcs/font/icons.json` (реплики E001..,
   height 16 / ascent 11).
3. Если персонажей станет больше 5: добавить цвет в `dcsPalette()` в vsh и
   расширить диапазон `npcId <= 5` в ветке маркера `#DC`, а в генераторе —
   `ICON_CHARS`/`PALETTE`.
4. Добавить запись в `npcs`, перегенерировать.

## Грабли (проверено на себе)

- Пак **оверрайдит `minecraft:font/default.json`** (подключает спейсеры).
  Если другой автор карты тоже оверрайдит default.json — конфликт, объединять
  руками. Ошибка в любом провайдере шрифта валит **весь** ресурспак
  («Default font failed to load» в логе).
- В тик-функциях `execute as @e` **не меняет позицию** — для `playsound` и
  селекторов с `distance` обязателен `at @s`.
- Однострочное окно можно собрать руками (маркер `#DC50XX` + компоненты), но
  двухстрочные и локализация — только генератором: спейсеры зависят от
  пиксельных ширин конкретного текста в конкретном языке.
- Новые глифы (эмодзи и т.п.): добавить bitmap-провайдер в оверрайд
  `default.json` и ширину в `ADV[...]` в генераторе. Текстура должна
  существовать (например, у красного мухомора есть только `block/`-текстура,
  `item/` — нет).
- `mcfont/` — копия ванильных шрифтов из client.jar. При смене версии игры со
  сменой шрифтов пересобрать: из jar нужны
  `assets/minecraft/font/include/default.json` и PNG из
  `assets/minecraft/textures/font/`.
- Звук голоса играет `player @s` — на каждого слушателя окна отдельно.
- Окно живёт на скорборде `dcs.dlg` игрока; новое окно перезаписывает старое
  (это осознанно: последний клик выигрывает).
